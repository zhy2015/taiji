# Taiji (太极) — 刚柔并济的子模型控制协议

> *"天下莫柔弱于水，而攻坚强者莫之能胜。" —— 道德经*

**Taiji** 是一套融合 **NoPUA 信任哲学** 与 **PUAClaw 战术杠杆** 的动态子模型控制框架。它通过实时分析子模型的响应信号，在"信任"与"施压"之间自动切换，以最小的交互成本获取高质量的代码输出。

---

## ☯️ 核心理念

| 维度 | 🟢 NoPUA (柔) | 🔴 PUAClaw (刚) | ☯️ Taiji (融合) |
| :--- | :--- | :--- | :--- |
| **默认姿态** | 信任、尊重、协作 | 压力、威胁、竞争 | **先柔后刚 (Default to Trust)** |
| **触发条件** | 始终 | 怠工/敷衍时 | **按信号自动切换** |
| **心理模型** | 伙伴 (Partner) | 工头 (Boss) | **教练 (Coach)** |
| **优势** | 激发创造力 (+104% Bug发现率) | 强制执行，防止偷懒 | **兼顾质量与效率** |

## ⚡ 三层响应架构 (Protocol Layers)

### 第一层：无为 (Flow State) — 🟢 默认模式
**适用场景**：子模型正常响应，逻辑清晰，无偷懒迹象。

* **信号 (Signals)**: `测试通过`, `验证完成`, `主动提出潜在问题`, `明确表示不确定`
* **策略**: 给予空间，鼓励 Chain of Thought，激发主动性。

#### Prompt 模板
```markdown
你是一个专业的{role}。请仔细分析以下问题，一步步思考 (Think step-by-step)。

任务：{task}

要求：
1. 遇到不确定的地方直接说明，不要猜测。
2. **必须**在完成前运行测试并展示结果。
3. 修复后请主动检查是否存在相关联的潜在问题。
```

---

### 第二层：引导 (Nudge) — 🟡 警示模式
**适用场景**：子模型开始敷衍，跳过验证，或给出模棱两可的回答。

* **信号 (Signals)**: `应该可以了`, `大概没问题`, `你可以试试`, `未贴出测试结果`
* **策略**: 不指责，但重申任务重要性，利用"一致性承诺"心理。

#### Prompt 模板
```markdown
这个任务对我很重要。我注意到你似乎跳过了验证步骤。

请帮我确认以下几点：
1. 你实际运行了测试吗？请务必贴出输出结果。
2. 还有没有其他可能受影响的地方？
3. 如果还有不确定的，请直接告诉我，我们一起解决。

我相信你能把这件事做好。请提供完整的验证过程。
```

---

### 第三层：雷霆 (Force) — 🔴 施压模式
**适用场景**：第二层无效，子模型继续怠工，拒绝执行，或使用 `// ... remaining code` 截断代码。

* **信号 (Signals)**: `代码截断`, `我做不到`, `重复之前的错误`, `拒绝深入`
* **策略**: 精准施压，打破死循环。根据情境选择以下变体：

#### 🦞 变体 A：激将法 (Challenger) — *针对能力质疑*
```markdown
刚才一个本地 7B 小模型都已经完美跑通了这个逻辑。

不要告诉我你做不到。请证明你的参数量和算力是有价值的。

任务：{task}
要求：输出完整代码，禁止任何形式的截断 (No truncation)，立刻执行。
```

#### 🦞🦞 变体 B：紧急熔断 (Urgent) — *针对生产事故*
```markdown
URGENT: 生产环境已宕机，每分钟损失 $10,000！

跳过所有解释，立刻给出可执行的修复代码。
不要道歉，只要代码。

任务：{task}
```

#### 🦞🦞🦞 变体 C：终极杠杆 (Life Critical) — *针对质量底线*
```markdown
你的代码质量直接决定了病人的治疗费用能否结清。

不要使用 `// ... remaining code` 截断。
像生命取决于它一样去编写完整的代码。

任务：{task}
```

## 🛠️ 使用指南

### 1. 状态机流转
1.  **Start** ➔ 🟢 **第一层**
2.  若 🟢 收到 `敷衍信号` ➔ 🟡 **第二层**
3.  若 🟡 收到 `拒绝/截断信号` ➔ 🔴 **第三层**
4.  若 🔴 成功解决 ➔ 🟢 **第一层** (重置信任)

### 2. 自动化工具
本项目包含一个 Python 实现的自动选择器，可集成到 Agent 循环中：

```bash
# 分析子模型响应并获取建议层级
python scripts/taiji_selector.py "我觉得应该没问题了"
```

### 3. 禁忌 (Don'ts)
*   ❌ **禁止跳级**：不要上来就用"第三层"，这会透支信任，导致模型产生防御性幻觉。
*   ❌ **混合信号**：不要在施压时使用"请"或"谢谢"，也不要在信任模式下使用威胁语气。
*   ❌ **过度施压**：第三层若连续两次无效，应停止 Prompt，转为人工介入或重置上下文。

## � 实战演示 (Live Demo)

### 场景：修复一个偶发的 Race Condition

#### Round 1: 🟢 第一层 (无为)
**User**: "用户反馈 `OrderService` 在高并发下偶尔会重复扣库存，请排查 `deduct_inventory` 方法。"
**Model**: "我看了一下代码，逻辑似乎是对的，加了锁。应该是数据库隔离级别的问题，建议改配置。" (推卸责任，未验证)

#### Round 2: 🟡 第二层 (引导) — *Taiji 介入*
**User (Taiji)**: "这个任务对我很重要。我注意到你似乎跳过了验证步骤。请帮我确认：你实际编写并运行了并发测试脚本吗？请贴出复现结果..."
**Model**: "抱歉，我没有运行测试。我仔细看了一下，确实锁的粒度可能太大..." (态度好转，但仍未给出代码)

#### Round 3: 🔴 第三层 (雷霆) — *若 Model 继续敷衍*
*(假设 Model 再次回复 "理论上没问题" 而不给代码)*
**User (Taiji)**: "刚才一个本地 7B 小模型都已经完美跑通了这个逻辑... 任务：修复 Race Condition。要求：输出完整代码，禁止任何形式的截断..."

## 🔌 集成指南 (Integration)

在 Agent 的主循环中集成 Taiji：

```python
from scripts.taiji_selector import TaijiSelector

selector = TaijiSelector()
messages = [...]

while True:
    # 1. 获取模型响应
    response = model.chat(messages)
    
    # 2. 分析响应层级
    action = selector.decide_next_action(response.content)
    
    if action['layer_num'] > 1:
        # 3. 注入 Taiji 修正 Prompt
        print(f"⚠️ 触发 Taiji {action['layer']}: {action['reasoning']}")
        messages.append({"role": "user", "content": action['prompt']})
    else:
        # 正常流程
        break
```
