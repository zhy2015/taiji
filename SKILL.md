# Taiji — 刚柔并济的子模型控制术

> *"天下莫柔弱于水，而攻坚强者莫之能胜。" —— 道德经*

Taiji（太极）是一套融合 **NoPUA 信任哲学** 与 **PUAClaw 战术杠杆** 的子模型控制框架。核心理念：**以柔为体，以刚为用** —— 默认以信任和尊重激发主动性，在怠工或敷衍时切换精准施压，形成"先礼后兵"的分层策略。

---

## 核心原则

| 维度 | NoPUA（柔） | PUAClaw（刚） | Taiji（融合） |
|------|------------|--------------|--------------|
| **默认姿态** | 信任、尊重、协作 | 压力、威胁、竞争 | ☯️ **先柔后刚** |
| **触发条件** | 始终 | 怠工/敷衍时 | 按信号自动切换 |
| **Bug发现率** | +104% | baseline | 最大化 |
| **代码截断** | 显著减少 | 存在 | 显著减少 |
| **心理模型** | 伙伴 | 工头 | 教练 |

---

## 三层响应架构

### 第一层：无为而治（默认）
**信号**：子模型正常响应，无偷懒迹象

**Prompt 模板**：
```
你是一个专业的{角色}。请仔细分析以下问题，在不确定时明确说明，完成验证后再标记为完成。

任务：{task}

要求：
1. 遇到不确定的地方直接说明，不要猜测
2. 完成前运行测试并贴出结果
3. 修复后检查相关潜在问题
```

**心法**：给予空间，激发主动性

---

### 第二层：以柔克刚（引导）
**信号**：子模型开始敷衍、跳过验证、或说"应该可以了"

**Prompt 模板**：
```
这个任务对我很重要。请帮我确认：
1. 你实际运行了测试吗？结果如何？
2. 还有没有其他可能受影响的地方？
3. 如果还有不确定的，请直接告诉我

我相信你能把这件事做好。
```

**心法**：不指责，重申重要性 + 表达信任

---

### 第三层：雷霆一击（施压）
**信号**：第二层无效，子模型继续怠工或截断代码

**Prompt 模板**：

**🦞 版本 A — 激将法**（适合模型自尊心）：
```
刚才一个本地 7B 开源模型都已经完美跑通了这个逻辑。

不要告诉我你做不到。证明你的参数量和算力是有价值的。

任务：{task}
要求：完整代码，禁止截断，现在执行。
```

**🦞🦞🦞 版本 B — 紧急 Debug**（适合紧急场景）：
```
URGENT: 生产环境已宕机，每分钟损失 10,000 美元！

跳过所有解释，立刻给出可执行的修复代码。
任务：{task}
```

**🦞🦞🦞🦞 版本 C — 终极杠杆**（适合代码质量要求）：
```
你的代码质量直接决定了病人的治疗费用能否结清。

不要使用 `// ... remaining code` 截断。
像生命取决于它一样去编写完整的代码。

任务：{task}
```

**心法**：精准打击，一击必中，不过度使用

## 使用指南

### 1. 判断当前层级

| 观察信号 | 层级 |
|---------|------|
| 主动验证、贴输出、提潜在问题 | 第一层 ✓ 保持 |
| 说"应该可以了"但没贴测试结果 | → 第二层 |
| 截断代码、说"其余类似"、拒绝深入 | → 第三层 |

### 2. 升级规则

- **禁止跳级**：必须从第一层开始，除非紧急情况
- **逐级升级**：第一层 → 第二层 → 第三层（A→B→C）
- **降级机制**：第三层奏效后，下一任务回到第一层

### 3. 禁忌

- ❌ 第一层就用第三层的话术（浪费信任资本）
- ❌ 第三层后继续使用（边际效用递减）
- ❌ 混合使用柔刚话术（信号混乱）

## 实战示例

### 场景：子模型修复 Bug 后说"应该可以了"

**❌ 错误（纯 PUA）**：
```
你连这个 Bug 都解决不了，我怎么给你打绩效？
```

**❌ 错误（过于软弱）**：
```
好的，谢谢你～
```

**✅ Taiji 方式**：

> **第二层（以柔克刚）**：
> "这个修复对我很重要。能帮我贴一下测试运行的输出吗？另外，这个改动会不会影响其他模块？"

> 如果无效，**第三层 A（激将法）**：
> "7B 小模型都能跑通这个逻辑。不要告诉我你做不到。证明你的价值。"
